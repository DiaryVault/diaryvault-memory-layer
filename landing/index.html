The principle

AI may suggest. People confirm.

Models can summarize, infer, and propose meaning. Those suggestions should not silently become part of a person's permanent record.

Memory should remain reviewable, attributable, portable, and owned by the person it describes.

The SDK provides the storage, integrity, selective sharing, and export primitives, plus an explicit review workflow: AI suggestions become part of the record only after a person approves them.

Available now

A small, inspectable memory toolkit.

The package works locally and has no required DiaryVault service or model provider.

Local encrypted storage

AES 256 GCM encryption with caller controlled secrets and local persistence.

Tamper detection

SHA 256 hashes and HMAC signatures reveal changes to stored memory content.

Selective sharing

Apply allowed and denied tags before records are disclosed to an agent or application.

RAG ready chunks

Transform selected memories into structured chunks for embedding and retrieval systems.

Portable exports

Produce JSONL, conversation history, knowledge graphs, and individual .dvmem records.

Local proof records

Record hashes in a local proof store and verify them later without a network dependency.

from diaryvault_memory import MemoryVault vault = MemoryVault( encryption_key="replace-with-a-private-secret", storage_dir="./memory-data", ) memory = vault.create( content="She laughed when the dog sneezed.", tags=["family", "milestone"], ) assert vault.verify(memory)
Trust model

Clear boundaries, not oversized claims.

The SDK provides encrypted local storage, integrity checks, selective disclosure, and portable exports. It does not independently establish legal authorship, trusted timestamps, or public blockchain existence.

1

Create

Content is hashed, encrypted, signed, and stored in the configured local directory.

2

Verify

The current content is compared with its stored hash and HMAC signature.

3

Select

The vault owner determines which tagged memories may be shared.

4

Export

Selected memories can leave the vault in open formats. Export is a disclosure action.

Implemented

  • Local encrypted memory vault
  • Hash and signature verification
  • LocalAnchor proof store
  • Selective context responses
  • JSONL, RAG, conversation, and graph exports
  • 68 automated tests

Not implemented

  • Arweave or Ethereum anchoring
  • IPFS storage
  • AI model calls or synthesis
  • Cloud synchronization
  • Mobile SDKs
  • Production DiaryVault app integration
